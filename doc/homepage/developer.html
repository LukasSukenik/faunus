Developer Information

The source code is documented using Doxygen and is available from the sidebar. Code additions are welcome: short contributions should be submitted to the ticket system (see link to the right), while if you want to become a regular contributer you may apply for repository write access.

Guidelines

Writing Code

  • Document code using Doxygen tags
  • Generously use C++'s assert() to ease debugging
  • Use a white space indentation of two when editing

Design

Good books on C++ design are C++ Coding Standards by Sutter and Alexandrescu as well as Effective C++ by Meyers. Some basic tips:

  1. Recycle code with polymorphic designs
  2. Use compile-time polymophism (templates) for speed limiting steps
  3. Exploit STL libraries and C++11 features
  4. Hide data and functions as much as possible (i.e. make them private)
  5. Ensure const-correctness
  6. Treat compiler warnings as errors

Committing

Before submitting to the repository, make sure nothing is broken: make all
make test Also verify SWIG modules: cmake -DENABLE_SWIG=on
make _pyfaunus
make test
