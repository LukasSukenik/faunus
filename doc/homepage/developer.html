Developer Information

The source code is documented using Doxygen and is available from the sidebar. Code additions are welcome: short contributions should be submitted to the ticket system (see link to the right), while if you want to become a regular contributer you may apply for repository write access.

Guidelines

  • Document code using Doxygen tags
  • Recycle code with object oriented designs
  • Generously use C++'s assert() for debugging
  • Hide data and functions as much as possible (i.e. private)
  • Treat compiler warnings as errors
  • Use white space indentation of two when editing
  • Before submitting to repository, make sure nothing is broken: make all
    make test     You should also make you the SWIG modules compile: cmake -DENABLE_SWIG=on
    make _pyfaunus
